# RAG-система для извлечения данных из документов
**Автор:** Виноградова Анна, группа 25.М81-мм 

## Описание
Реализует Retrieval-Augmented Generation (RAG) систему, которая:
- ищет релевантные документы в векторном хранилище (FAISS);
- фильтрует их по компаниям, упомянутым в вопросе;
- дополнительно фильтрует документы по компаниям, упомянутым в вопросе;
- формирует контекст и отправляет запрос в LLM (DeepSeek);
- возвращает ответ строго в формате JSON с указанием источника (чанка);
- автоматически формирует `submission.json` и отправляет его на сервер.

## Описание основных компонентов 
### `config.py` 
Содержит: 
- пути к файлам и директориям; 
- путь к FAISS-индексу; 
- API-ключ для доступа к DeepSeek. 
>  API-ключ необходимо указать вручную. 

### `extract_companies.py` 
- выполняет выделение наименований компаний с помощью NER (spaCy); 
- нормализует их; 
- сохраняет уникальный список компаний в `data/companies`. 

### `vector_store.py` 
- загружает PDF-документы; 
- разбивает их на чанки; 
- определяет, какие компании упоминаются в каждом чанке; 
- сохраняет информацию о компаниях в метаданные; 
- строит и сохраняет FAISS-векторное хранилище в `data/rag_index`. 
--- 
### `main.py` 
Основной исполняемый скрипт, который: 
1. загружает вопросы; 
2. извлекает компании из каждого вопроса; 
3. выполняет поиск релевантных чанков в FAISS, в том числе с помощью извлеченных компаний; 
4. формирует контекст для LLM; 
5. отправляет запрос в модель DeepSeek; 
6. валидирует и приводит ответы к требуемому типу; 
7. формирует `submission.json`; 
8. отправляет файл на сервер.
